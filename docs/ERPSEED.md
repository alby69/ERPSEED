## 1. Visione e Missione

### 1.1 La Nostra Visione

**ERPSeed** è un ecosistema tecnologico partecipativo che democratizza l'accesso alla tecnologia ERP e alla formazione informatica, creando opportunità concrete di crescita professionale per giovani in contesti locali, inclusi quelli delle zone più disagiate.

> **Il nostro sogno**: Un mondo in cui ogni microimpresa, anche nella più piccola frazione di provincia, possa avere accesso a strumenti digitali professionali - e in cui ogni giovane possa imparare a crearli.

### 1.2 Missione

| 🎯 Obiettivo | 📝 Descrizione |
|--------------|----------------|
| **Democratizzare la tecnologia** | Permettere a microimprese e persone in zone svantaggiate di accedere a strumenti ERP professionali a costi irrisori |
| **Formare nuove generazioni** | Creare percorsi di crescita professionale per giovani studenti, universitari e professionisti in cerca di riqualificazione |
| **Costruire comunità** | Sviluppare una rete globale di nodi autosufficienti che si supportano reciprocamente |
| **Open Source** | Mantenere tutto il codice aperto e accessibile, rifiutando lock-in vendor |

### 3.3 Dettaglio Componenti

#### 🌱 FlaskERP (Prodotto)
Il cuore tecnologico del progetto:
- **ERP low-code** costruibile via browser
- **AI Assistant** per generazione automatica configurazioni
- **Marketplace** per moduli creati dalla community
- **Multi-tenant** per servire molteplici clienti
- **Open source** (MIT License)

#### 📍 Nodi Locali
Gruppi fisici distribuiti geograficamente:
- 3-15 membri attivi
- Incontri regolari (settimanali/bi-settimanali)
- Progetti per clienti locali
- Autogestione economica
- Governance democratica interna

#### 🖥️ Piattaforma Web
L'infrastruttura digitale che connette tutto:
- Sistema centralizzato di autenticazione
- Gestione profili e progressione
- Repository codice (GitHub integration)
- Area formativa (corsi, certificazioni)
- Forum e chat
- Mappa nodi globali

### 4.3 Modello Multi-Tenant

I nodi e i loro progetti clienti vivono in spazi separati:

```python
# Schema multi-tenant per nodi
class Node(BaseModel):
    id: int
    name: str
    location: str          # Città
    country: str           # Paese
    founded_date: date
    status: enum           # active, paused, closed

class NodeMember(BaseModel):
    node_id: int
    user_id: int
    role: enum             # founder, senior, junior, apprentice
    joined_date: date
    contribution_hours: float

class Project(BaseModel):
    node_id: int
    client_name: str
    flaskerp_instance: str   # Subdomain o path
    status: enum
    created_date: date
```
